# IBM Internship Technical Task: Comment-to-Ticket Triage

This repository contains a backend application for the IBM "Comment-to-Ticket Triage" internship challenge. The system allows users to submit comments via REST API, which are then analyzed using a Hugging Face AI model to determine if a support ticket should be created. If a ticket is warranted, it is automatically categorized and prioritized with a summary generated by the AI.

🚀 **Live Demo:** [https://ibm-task-comtotic-450725227122.europe-west1.run.app/](https://ibm-task-comtotic-450725227122.europe-west1.run.app/)

---

## Features

- **Comment Submission & Viewing**: Submit user feedback via API and list all received comments.
- **AI-Powered Ticket Generation**: Comments are automatically analyzed using the Hugging Face Inference API. If the comment describes an issue or request, a support ticket is generated, categorized, and prioritized.
- **Ticket Management**: View all generated tickets, as well as details for each ticket.
- **In-Memory / Embedded Database**: Comments and tickets are persisted using an H2 embedded database (default).
- **RESTful API** built with Java + Spring Boot.
- **(Optional Bonus)** Simple HTML UI to submit comments and view tickets.

## API Endpoints

| Method | Endpoint          | Description                         |
|--------|-------------------|-------------------------------------|
| POST   | `/comments`       | Submit a new comment                |
| GET    | `/comments`       | List all comments                   |
| GET    | `/tickets`        | List all AI-generated tickets       |
| GET    | `/tickets/{id}`   | View details for a specific ticket  |

---

## How the AI Works

Upon comment submission, the backend sends the comment to a Hugging Face model (e.g. [`google/flan-t5-base`](https://huggingface.co/google/flan-t5-base)) via the Inference API.

The model determines:
- Should this comment open a support ticket?
- If yes, it generates:
  - **Title** — a short descriptive title
  - **Category** — `bug` / `feature` / `billing` / `account` / `other`
  - **Priority** — `low` / `medium` / `high`
  - **Summary** — a brief description of the issue

Tickets are stored in the H2 database and accessible via the `/tickets` endpoints.

## Setup & Run

### 1. Clone the Repository

```bash
git clone https://github.com/VolodymirM/IBM-internship-task.git
cd IBM-internship-task
```

### 2. Configure the Hugging Face API Key

Obtain a free API token from [Hugging Face](https://huggingface.co/settings/tokens) and add it to `src/main/resources/application.properties`:

```properties
huggingface.api.key=hf_your_token_here
```

Or set it as an environment variable:

```bash
export HUGGINGFACE_API_KEY=hf_your_token_here
```

### 3. Build & Run

Using Maven:

```bash
./mvnw spring-boot:run
```

Using Gradle:

```bash
./gradlew bootRun
```

The server starts at `http://localhost:8080`.

## API Usage Examples

**Submit a comment:**

```bash
curl -X POST http://localhost:8080/comments \
     -H "Content-Type: application/json" \
     -d '{"content": "My login is broken. I get an error every time I try to sign in."}'
```

**List all comments:**

```bash
curl http://localhost:8080/comments
```

**List all tickets:**

```bash
curl http://localhost:8080/tickets
```

**View a specific ticket:**

```bash
curl http://localhost:8080/tickets/1
```

## Run with Docker (Optional)

**Build the image:**

```bash
docker build -t ibm-internship-task .
```

**Run the container:**

```bash
docker run -e HUGGINGFACE_API_KEY=hf_your_token_here -p 8080:8080 ibm-internship-task
```

The app will be available at `http://localhost:8080`.
